# 简历项目描述
 —— 智能广告投放助手

## 一句话定位
基于 DeepSeek API + Function Calling 的 AI Agent 广告投放管理平台

## 技术栈
后端：Flask / Python / SQLite / JWT
AI：DeepSeek API / Function Calling / Tool Registry / 树状记忆系统
前端：原生 HTML + CSS + JavaScript + ECharts
测试：Python 单元测试（14 个用例）
工具：Git / dotenv

## 项目描述（150字版，简历用）

基于 Flask + DeepSeek API 开发的智能广告投放助手，实现了完整的 AI Agent 架构。设计了 24 个函数工具（仪表盘/预算/告警/趋势分析等），通过 Function Calling 让 LLM 自动识别用户意图并调用对应函数。实现树状记忆系统，按平台账户/最近操作/告警/偏好四个分支存储用户历史，自动学习用户偏好。三级权限系统（boss/admin/user），JWT 认证，14 个单元测试覆盖核心功能。

## 项目亮点（面试展开讲）

1. **完整 Agent 架构** — 从提示词设计 → LLM 调用 → tool_calls 判断 → 函数执行 → 结果返回，全闭环手写实现（未使用 LangChain 等框架）

2. **24 个函数工具** — 覆盖广告投放全场景：仪表盘核心指标、每日趋势、预算管理、告警检测、计划状态控制、热销产品查询、搜索趋势分析等

3. **树状记忆系统** — 按 4 个维度（平台账户/最近操作/告警与问题/关注偏好）分层存储用户历史操作，自动学习用户偏好（如频繁查看仪表盘→推导"关注ROAS"），每次对话注入提示词

4. **安全边界设计** — 通过 Prompt 规则（6 条）约束 AI 行为：敏感操作只建议不执行、数据不足时不编造、不暴露内部过程

5. **三级权限 + 数据隔离** — boss/admin/user 角色权限，每个角色看到不同数据，操作日志按角色过滤

6. **单元测试** — 3 个测试文件 14 个用例覆盖仪表盘/登录/工具函数

### 我的收获（通过这个项目学到了什么）

- **Agent 工作原理**：深入理解了 LLM 不是"万能回答机"，而是通过 Function Calling + 工具注册表 + Prompt 约束才能变成一个有用的助手
- **Function Calling 本质**：本质上就是 LLM 做语义匹配 → 算概率 → 选函数 → 调用的流程，跟传统 if-else 的区别在于 LLM 能理解自然语言和错别字
- **记忆管理**：理解了长期记忆不能靠无限堆对话历史，而是需要结构化存储 + 摘要压缩，树状结构比平面压缩更准确
- **Prompt 工程**：好 prompt 不是一次写成的，是在迭代中逐步加规则调出来的，每条规则解决一个实际出现过的问题
- **工程思维**：模块化解耦、单元测试、文档建设不是可有可无的，是保证项目能持续维护的基础
