## 二、核心流程

### 2.1 用户请求处理流程

```
用户输入 → 路由层 → Agent 层 → LLM → 工具层 → 数据层 → 返回
                    ↓
              记忆层注入
              (树状记忆+总结)
```

### 2.2 AI 对话流程

```
用户: "今天数据怎么样"
   │
   ▼
1. chat_routes.py 保存用户消息
   │
   ▼
2. agent.chat() 读取记忆 + 历史
   │  ┌──────────────────────────┐
   │  │ 注入【长期记忆】到提示词   │
   │  │ 注入【最近15条】对话历史   │
   │  └──────────────────────────┘
   │
   ▼
3. 调用 DeepSeek API (带25个函数描述)
   │
   ▼
4. AI 返回 tool_calls → 调用 get_dashboard_data()
   │
   ▼
5. 执行函数，获取数据
   │
   ▼
6. 更新树状记忆 + 检查是否需要自动总结
   │
   ▼
7. 再次调用 API (让AI组织语言)
   │
   ▼
8. 返回: "今天总花费22400，ROAS 2.47，表现正常"
```

### 2.3 RAG 检索流程

```
用户: "ROAS多少算正常"
   │
   ▼
1. AI 识别需要外部知识 → 调用 search_knowledge()
   │
   ▼
2. 向量检索 (TF-IDF + 余弦相似度)
   │  ┌──────────────────────────┐
   │  │ 知识库 12 条 → 向量化     │
   │  │ 用户问题 → 向量化         │
   │  │ 算相似度 → 取 Top 3      │
   │  └──────────────────────────┘
   │
   ▼
3. 返回: "行业 ROAS 2-4 为良好"
   │
   ▼
4. AI 结合数据回答: "你的 ROAS 2.47，属于正常水平"
```

## 三、记忆系统架构

### 3.1 树状记忆 (5个分支)

```
user_{id}.json
│
├── 平台账户    ← 存账户余额 (30天过期)
├── 最近操作    ← 最近10条操作 (3天过期)
├── 告警与问题  ← 最近5条告警 (7天过期)
├── 关注偏好    ← 自动学习 (14天过期)
├── 阶段总结    ← 每5条自动生成 (30天过期)
│
├── stats      ← 操作统计 (用于偏好学习)
└── _last_summary_count ← 上次总结时的操作数
```

### 3.2 记忆注入 (get_compact_memory)

每次对话前，树状记忆被格式化为文字注入系统提示词：

```
【平台账户】Google Ads(余额18000)
【最近】[07-03] 仪表盘：花费22400 | [07-03] 搜索趋势：AI广告
【告警】[07-03] 预算即将用尽
【用户偏好】用户经常查看仪表盘，关注ROAS
【阶段总结】[07-03] 查看了2次仪表盘，处理了1条告警
```

## 五、技术栈与依赖

| 分类 | 技术 | 用途 |
|------|------|------|
| 后端框架 | Flask 3.x | HTTP服务 + 路由 |
| AI 接口 | DeepSeek API (OpenAI SDK) | LLM 推理 + Function Calling |
| 数据库 | SQLite | 结构化数据存储 |
| 向量检索 | scikit-learn (TF-IDF) | RAG 语义检索 |
| 认证 | PyJWT (HS256) | 用户身份验证 |
| 验证码 | Pillow | 登录验证码图片 |
| 前端 | 原生 HTML/CSS/JS + ECharts | SPA 单页应用 |
| 测试 | Python unittest (14个用例) | 质量保障 |
